reVX.reeds.reeds_cli module¶

reV-ReEDS¶

ReEDS Command Line Interface

reV-ReEDS [OPTIONS] COMMAND [ARGS]...

Options

-n, --name <name>¶

Job name. Default is “ReEDS”.

Default

ReEDS

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

from-config¶

Run reVX-REEDS from a config.

reV-ReEDS from-config [OPTIONS]

Options

-c, --config <config>¶

Required Filepath to reVX-REEDS config json file.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

local¶

Run reVX-REEDS on local hardware.

reV-ReEDS local [OPTIONS] COMMAND1 [ARGS]... [COMMAND2 [ARGS]...]...

Options

-o, --out_dir <out_dir>¶

Required Directory to dump output files

-log, --log_dir <log_dir>¶

Directory to dump log files. Default is out_dir.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

classify¶

Extract ReEDS (region, bin, class) groups

reV-ReEDS local classify [OPTIONS]

Options

-rt, --rev_table <rev_table>¶

Required Path to .csv containing reV aggregation or supply curve table

-rc, --resource_classes <resource_classes>¶

Required resource_classes: str | pandas.DataFrame Resource classes, either provided in a .csv, .json or a DataFrame Allowable columns: - ‘class’ -> class labels to use - ‘TRG_cap’ -> TRG capacity bins to use to create TRG classes - any column in ‘rev_table’ -> Categorical bins - ‘*_min’ and ‘*_max’ where * is a numberical column in ‘rev_table’ -> Range binning NOTE: ‘TRG_cap’ can only be combined with categorical bins

-r, --regions <regions>¶

Mapping of supply curve points to geographic region

Default

reeds_region

-cb, --cap_bins <cap_bins>¶

Number of capacity bins to create for each region/resource bin combination

Default

5

-sb, --sort_bins_by <sort_bins_by>¶

Column(s) in rev_table to sort before binning

Default

trans_cap_cost

-f, --pre_filter <pre_filter>¶

Column value pair(s) to filter on. If None do not filter

-tbr, --trg_by_region¶

Groupby on region when computing TRGs

profiles¶

Extract ReEDS represntative profiles

reV-ReEDS local profiles [OPTIONS]

Options

-rt, --reeds_table <reeds_table>¶

Path to .csv containing reeds classification table not needed if chained with classify command

-cf, --cf_profiles <cf_profiles>¶

Required Path to reV .h5 file containing desired capacity factor profiles

-gc, --gid_col <gid_col>¶

Column label in rev_summary that contains the generation gids (data index in cf_profiles file path).

Default

gen_gids

-np, --n_profiles <n_profiles>¶

Number of profiles to extract per “group”.

Default

1

-pd, --profiles_dset <profiles_dset>¶

Profiles dataset name in cf_profiles file.

Default

cf_profile

-rm, --rep_method <rep_method>¶

Method identifier for calculation of the representative profile.

Default

meanoid

-em, --err_method <err_method>¶

Method identifier for calculation of error from the representative profile.

Default

rmse

-w, --weight <weight>¶

Column in rev_summary used to apply weighted mean to profiles. The supply curve table data in the weight column should have weight values corresponding to the gid_col in the same row.

Default

gid_counts

-rcp, --reg_cols <reg_cols>¶

Label(s) for a categorical region column(s) to extract profiles for (default is region, bin, class)

Default

region, bin, class

-mw, --max_workers <max_workers>¶

Number of parallel workers. 1 will run serial, None will use all available.

timeslices¶

Extract timeslices from representative profiles

reV-ReEDS local timeslices [OPTIONS]

Options

-pr, --profiles <profiles>¶

Path to .h5 file containing (representative) profiles, not needed if chained with profiles command

-ts, --timeslices <timeslices>¶

Required .csv containing timeslice mapping

-rct, --reg_cols <reg_cols>¶

Label(s) for a categorical region column(s) to create timeslice stats for (default is region and class)

Default

region, class

-ap, --all_profiles¶

Flag to calculate timeslice stats from all CF profiles as opposed to representative profiles (default).

valid-config-keys¶

Echo the valid ReEDS config keys

reV-ReEDS valid-config-keys [OPTIONS]
